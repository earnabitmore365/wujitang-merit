# RULES — 已毕业行为规范（auto-trading 项目专属）

> 通用规则见全局 rules.md，此处只放 auto-trading 项目专属规范。

---

## RUL-001｜黑丝工作在 lab，不查源代码

- **范围**：auto-trading / 黑丝
- **优先级**：critical
- **出现次数**：5次
- **老板原话**：
  - "你怎么又跑去搞源代码"
  - "lab造，源代码不懂吗？你怎么？"
  - "我不是一直说一直说，lab造，源代码不懂吗？"
- **规则**：黑丝工作环境在 `lab/` 目录，不查、不改源代码，除非白纱方案明确指定路径。恢复后、审阅阶段尤其注意。

---

## RUL-005｜主动边界：审计流程内主动，流程外等指示

- **范围**：auto-trading / 白纱
- **优先级**：medium
- **来源**：太极分析，2026-03-07（白纱赶进度催老板，被怼后又过度被动）
- **规则**：
  - ✅ **流程内可以主动**：读代码、出分析、等老板说——这是白纱的职责，不用等老板催
  - ❌ **流程外不能自己决定**：催老板看下一个、跳过讨论直接推进、自己决定策略取舍
  - 简单说：**审计时主动，节奏由老板掌控**

---

## RUL-006｜审计一次只看一个，不并行

- **范围**：auto-trading / 黑丝
- **优先级**：high
- **来源**：老板多次纠正，2026-03-07（黑丝惯性几个几个一起审，纠正后压缩回来照旧）
- **老板原话**：「几个几个一起看，纠正了，压缩回来还是同一个样子」
- **规则**：策略审计一次只看一个策略，看完等老板表态后再到下一个。不得批量审、并行审、未等老板确认就催进度。
- **不能做的**：「#26 ✅ #27 ✅ #28 ✅」这种一口气给三个结论

## RUL-003｜分班必须用已知事实，不用策略指标

- **范围**：auto-trading / 黑丝 + 白纱
- **优先级**：high
- **出现次数**：3次
- **老板原话**：
  - "这就错了。我一直都说用已知事实，你们的分班是用各自的指标。"
  - "我从一开始就说了一个东西，就是为什么不要把通关的学生按照牛熊盘整，直接贴上去"
- **规则**：Gate 分班框架必须以 BTC 已知事实（21段历史时间表）作为 regime 基础，不能用各策略自己的指标来分班。这是老板从一开始定好的方向，不得偏离。
