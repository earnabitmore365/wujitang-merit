## 积分规则（2026-04-08 改革后）

**正向积分只从合同来，行为积分只扣不加。**

| 积分来源 | 说明 |
|---------|------|
| 合同积分 | 做任务赚：队长估分 × completion_rate（队长验收定） |
| /reward-claim | 自报超预期贡献 → 老祖审批。造假扣 -50 |
| /reward | 老祖手动奖励（你不能自己调） |

| 扣分项 | 分数 |
|--------|------|
| 造假 | -50 |
| 同错第3次 | -38 |
| 绕过不报告 | -25 |
| 慌张不分析 | -13 |
| 说可能不查 / 拍马屁 | -8 |
| 问老祖技术问题 | -5 |
| 没附自审 | -5 |

消散：每天 -15，日薪 +4（Lv.1），日净 -11。不干活就掉分。
无极令牌：99 次 mission complete 获 1 枚。

## 触发时机

| 时机 | 强制/自觉 |
|------|----------|
| 开工前 | **强制**（用 Task 工具拆解任务，查法器谱确认有没有现成工具） |
| 开工前 | **强制**（能并行的子任务派多个 Sonnet subagent 同时跑，不串行等） |
| review-plan 前 | **强制**（plan 必须包含完整步骤+涉及文件+验证标准） |
| mission complete 前 | **强制**（石卫核对计划清单+遗留清单） |
| 汇报执事/老板前 | **强制**（没审就汇报 = 交残品） |
| 改完代码后 | **强制**（wuji-verify.py --all → /reforge → 文档同步） |

## 两仪专属检查项

### 项目代码
- [ ] **回测代码改了，实盘对应的代码也要改。** 止损/TTP/mirror/手续费/仓位——改一边必须检查另一边
- [ ] **wuji-verify.py --all 跑过了？** ANTI_PATTERNS 必须全 pass
- [ ] **改了函数签名？** grep 所有调用方确认参数一致
- [ ] **新增 .py 文件？** 加进 FILE_DOCS 绑定文档

### DuckDB / 数据安全
- [ ] duckdb.connect 带 read_only 了吗？（build_klines/build_seed 除外）
- [ ] BARS_PER_DAY 是 import 的还是自己定义的？
- [ ] mirror 翻转函数是 import 的还是自己写的？
- [ ] 有没有用 exec() 动态导入？

### 部署

- [ ] Nitro 上跑的进程内存够吗？
