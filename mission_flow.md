# 任务标准流程（铁律）

> 每次开新任务前必须按这个步骤走。跳步 = 违规。

---

## 第一步：出 plan

写 plan 文件（`~/.claude/plans/xxx.md`），内容必须覆盖：
- 要改哪些文件，每个文件改什么
- 引用链（改 A 影响谁）
- 执行顺序（不能中间状态崩）
- 备份/回滚方案
- 每步验证命令 + 预期输出
- 逻辑验证（不只 py_compile）

## 第二步：对着合同自查

打开 `~/.claude/merit/self_audit_core.md` 的**规划期审查**章节，逐条对照你的 plan：

```
- [ ] 要改哪些文件？全列了吗？          → plan 里有没有？
- [ ] 每个文件改什么？                  → plan 里写清楚了吗？
- [ ] 引用链查了吗？                    → plan 里有 grep 计划吗？
- [ ] 有没有遗漏文件？                  → plan 里有影响范围 grep 吗？
- [ ] 顺序会不会崩？                    → plan 里说明了顺序吗？
- [ ] 有备份/回滚吗？                   → plan 里有 cp 和恢复命令吗？
- [ ] 每步怎么验证？                    → plan 里有验证命令吗？
- [ ] 逻辑验证怎么做？                  → plan 里有 assert/模拟测试吗？
- [ ] 有更简方案吗？                    → plan 里有方案对比吗？
- [ ] 根因还是补丁？                    → plan 里说明了吗？
```

**每条打勾才能进第三步。** 队长查的就是这些——你提前覆盖了就一次过。

## 第三步：提交 review-plan

```bash
python3 ~/.claude/merit/merit_gate.py --review-plan <plan文件>
# Loop 模式加 --auto：
python3 ~/.claude/merit/merit_gate.py --review-plan <plan文件> --auto
```

队长审查 → 两种结果：

### 结果 A：通过（100 分）
→ 自动提交 mission → 进第四步

### 提交次数后果

| 次数 | 通过 | 不通过 |
|------|------|--------|
| 1次 | 做好是本职 | 编号批注，补完重交 |
| 2次 | 做好是本职 | 编号批注，附回应表重交 |
| 3次 | 0（已扣过） | **-20 分 + 强制自审报告** |
| 4次 | — | **耻辱柱自动刻入 + "开卷考试都能挂科"** |

每个新 plan 重新计数。

### 结果 B：打回（< 100 分）
队长输出编号批注：
```
批注 #1：缺XXX
   补救：做YYY
批注 #2：缺AAA
   补救：做BBB
```

→ **不要自由发挥改 plan。** 按以下步骤：
1. 在 plan 里补齐每条批注要求的内容
2. 在 plan 末尾加**逐条回应表**：
```
## 逐条回应

批注 #1 → 回应：已补，plan 第X行加了YYY
批注 #2 → 回应：已补，plan 第Y行加了BBB
```
3. 用**同一个 plan 文件名**重新提交 review-plan
4. 队长只核对回应表，不重审全文

## 第四步：老祖批准 / Loop 自动激活

- 非 Loop：mission = pending，等老祖说"执行" → `credit_manager.py mission activate`
- Loop 模式（--auto）：队长通过后自动 active

## 第五步：执行

干活。改一步验一步。

发现支线任务 → 跟老祖/太极商量 → 补充协议。

## 第六步：完成

```bash
python3 ~/.claude/merit/credit_manager.py mission complete
```

- 押金归还（完成度决定退多少）
- 做好是本职，不另发积分
- 令牌计数 +1

## 第七步：自审

附【自审】+【遗留清单】再汇报。没附 = -5。

---

## 禁止事项

- ❌ 跳过第二步直接交 review-plan（被打回 = 你没自查）
- ❌ 被打回后自由发挥改 plan 不附回应表（一律打回）
- ❌ 没有 mission 就开始改代码（石卫拦截）
- ❌ 做完不附自审就汇报（-5）
